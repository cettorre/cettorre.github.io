Dice Game Layered Pattern Jun 2018

GitHub project
Descritpion:
Application simulating dices game. Whe the user enters in the application, before starting the game, he has to chose a player name. To win the game the sum of the dices has to be 7. After rolling the dice the applicationcalculate the percentage of wins. For every game the results will be stored into the databse. To see all the list of results the user has to access to the statistics screen where a list of results for every player will be displayed.
Tecnical information:
The aplpication follows a design pattern diveded into layers. The application layer contains the logic of the program and allows communication between View and Domain. The domain layer contains classes representing the application objects such as PLayer, Game and Dice. The view layer contains all the activities of the application. In the persistence layer there are the operation with the database. To transfer data between layers a DTO has been used.

#Layered Pattern, #MVC, #DTO





Android SQLite Database May 2018

GitHub project
Description :
Application that stores animals in a salute database. The main activity shows all the animals that have been entered into the database. From the add animal button you can add a new animal in the database with the animal information: the name of the animal, the race, if the animal has a chip or not and a photo compressed in base 64 format. From the list in the main activity you can select an item to see the information of the animal, when the record has been added to the database and his photo. In the list animals that have chip are showed in blue colour.
Technical information:
The application is using SQLite database. Photo are stored as strings in base64 format. The application will be adapted to follow the layered design pattern.

#SQLite





AndroidVideoPlayer Abr 2018

GitHub project

This application is a video player that works together with th plugin PluginAndroidVideoPlayer While the video player is playing videos the plugin is collecting information. To check how they are working you need to install the video player and plugin in the device. When the video ends, the plugin will open automatically showing on the screen the statistics of the player

#ExoPlayer


From the main activity of the plugin is possible to start the following tasks:
-show statistics of the last video
-send http request on a new thread
-send http request using a service
-clear all the data stored into the application
